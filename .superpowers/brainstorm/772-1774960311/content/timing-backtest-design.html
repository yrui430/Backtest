时序回测类 — 完整设计 (基于 read_me.md)

严格遵循 read_me.md 定义的流程

1. 数据输入层 — DataSpec
支持格式: h5, csv, parquet
路径 A: 有 Label
use_raw_data = False
输入: feature_df + label_df
用户指定: feature列名, label列名, time列名
→ 校验 timestamp 对齐
→ 不对齐则 print 中断
路径 B: 无 Label (原始数据)
use_raw_data = True
输入: feature_df + ohlcv_df
用户指定: time列名, close/vwap列名
exec_price_type: "close" | "vwap"
→ 内部计算 label (一阶差分)
→ 校验 timestamp 对齐
2. Label 定义 — LabelSpec
label_horizon: int = 1
label=1: 信号发出后下一个bar开仓, 持有1个bar
label=2: 持有2个bar, position各0.5叠加
label=5: 持有5个bar, 5个0.2仓位叠加
收益率 = Σ(各仓位收益), 每个仓位 = 1/label_horizon
lag: int = 1
lag=1: 出信号后下一个bar交易 (默认)
lag=2: 出信号后第二个bar交易
exec_price_type 决定成交价:
close → 用 close 价成交
vwap → 用 vwap 价成交
3. 信号生成 — SignalSpec
方法 A: 分位数法 (默认)
upper_quantile = 0.8
lower_quantile = 0.2
rolling_window = 100
feature > upper → long
feature < lower → short
方法 B: 阈值法
upper_threshold: float (绝对数字)
lower_threshold: float (绝对数字)
feature > upper → long
feature < lower → short
方法 C: 自定义映射
signal_mapper: Callable = None
用户 import 映射函数
启用时可选: 换仓 hurdle
默认 Equal Weighted
展示模式选择
display_mode: "long_only" | "short_only" | "long_short" | 任意组合 | "all"
4. 执行 & 费率
fee_rate: float = 0.00005 # 万五单边
show_before_fee: bool = True
show_after_fee: bool = True
hurdle_enabled: bool = False # 仅自定义映射时可用
hurdle_func: Callable = None # 自定义换仓hurdle
5. 输出 — 表格 + 图表 (数形结合)
表格字段
• Annualized Return
• Total Return
• Volatility
• Sharpe Ratio
• Sortino Ratio
• Turnover
• Max Drawdown
• Max Drawdown Recovery Time
• Rank IC
• IC
• IR (Information Ratio)
图表 (英文标注)
• PnL Curve (Before Fee)
• PnL Curve (After Fee)
• Rank IC Rolling Window
• IC Rolling Window
⚠ 禁止使用 pandas rolling 函数
⚠ 必须用 numba 加速实现
6. 关键约束 (Tips)
① timestamp 不对齐 → 中断
print 提示后中断, 不继续
② 不用 rolling 函数
IC 和 分位数 rolling 全部用 numba
③ 多 label 叠加仓位
label=2 → 2个0.5仓位, label=5 → 5个0.2仓位
④ 自定义映射 → hurdle 模块
无自定义映射 → 跳过 hurdle
